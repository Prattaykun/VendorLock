# Beat Plan API Endpoints

> This document details the endpoints related to **Beat Plan**, mapping them to business logic in `vendorlock.txt` and frontend components.

## `GET /api/v1/beat-plan/{salesman_id}`
**Get daily beat plan for a salesman**

Returns today's optimised beat plan for the given salesman. Generated by Agent 6.

- **Authentication / Roles:** Any Authenticated (Admin, Distributor, Salesman, Retailer)
- **AI Agents Involved:**
  - **Agent 6 ‚Äî Beat Intelligence & Coverage**: Optimises routes, detects ghost visits and coverage gaps.
  - *VendorLock Spec Reference*: Section: 4. Monitors Beat Coverage ‚Äî Catches Ghost Visits
  - *Setup*: Requires DB: `outlets`, `beat_checkins`, `salesmen`.
- **Frontend Components:** Generic Dashboard Component

### Response (Success)
```typescript
{
    beat_plan_id*: string
    salesman_id*: string
    date*: string
    route_name*: string
    total_stops*: integer
    stops*: Array<OutletStop>
    estimated_revenue_opportunity*: number
    ghost_risk_outlets*: Array<string>
}
```
